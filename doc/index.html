1. Installation

1

Install Typesense

You need a running Typesense server. You can self-host it using Docker (free) or use Typesense Cloud.

2

Install the Plugin

Upload the swift-search-typesense.zip file to your WordPress Plugins dashboard and activate it.

3

Setup Wizard

Go to the SwiftSearch menu in your admin dashboard. Enter your Typesense credentials (Node URL, API Key) to connect.

2. Configuration & Indexing

SwiftSearch simplifies index management with a robust dashboard.

API Keys Best Practice

For optimal security, we recommend using two separate API keys:

  • Admin Key: Used by the plugin to create schemas and index content securely from the backend.
  • Search-Only Key: Used by the frontend/browser to fetch results. This key is read-only and safe to expose.

Reliable Background Indexing

Click "Index All" to start building your search index. The plugin uses the industry-standard Action Scheduler (the same library used by WooCommerce) to process thousands of items in the background without slowing down your site.

4. Displaying the Search

To display the clean search bar anywhere on your site (header, sidebar, or inside a page), use the shortcode:

[swift_search]

Advanced Options

You can customize the search behavior using specific attributes:

  • placeholder: Custom placeholder text (Default: "Search...").
  • limit: Maximum number of results to display (Default: 10).
  • show_thumbnail: "true" to show images, "false" for text-only.
  • show_price: "true" to display WooCommerce prices.
  • show_excerpt: "true" to display post content snippets (Default: false).
  • instant_search: "true" (Overlay), "false" (Standard Form), or "default".
  • scope: "posts" (Articles/Products) or "terms" (Categories/Tags) or "users" (Authors).
  • post_types: Comma-separated list (e.g. "product,post").

Example Usage

[swift_search placeholder="Find products..." limit="8" post_types="product" show_price="true"]

6. Theming & Customization

The search interface is designed to inherit your theme's fonts, but you can override styles using standard CSS. The plugin uses a BEM-like naming convention.

Common Selectors

/* The Outer Wrapper */
.ss-wrapper { 
    max-width: 600px; /* Adjust width */
}

/* The Search Input Box */
.ss-search-box input { 
    border-radius: 8px;
    border: 2px solid #e5e7eb;
}

/* The Results Dropdown */
.ss-results-container { 
    box-shadow: 0 10px 25px rgba(0,0,0,0.1);
}

/* Result Items */
.ss-hit { 
    border-bottom: 1px solid #eee;
}

.ss-hit-title { 
    font-weight: bold;
    color: #111;
}

.ss-hit-thumb { 
    width: 60px; 
    height: 60px;
    border-radius: 4px; 
}

/* Highlighted Matches */
.ss-hit-title mark { 
    background-color: rgba(255, 0, 85, 0.1);
    color: #ff0055;
}

7. Developer Hooks (Actions)

You can interact with the indexing engine programmatically. This is useful for custom import scripts or 3rd-party integrations.

Async Indexing

The plugin uses Action Scheduler to process posts in the background. You can manually trigger a re-index for any post:

// Trigger re-index for Post ID 123 (Runs in background)
do_action('swift_search_async_index_post', 123);

8. Troubleshooting & FAQ

Result "Connection Failed"

  • Mixed Content: If your site uses HTTPS, your Typesense server MUST also use HTTPS (wss://).
  • CORS: Check your browser console (F12). If you see CORS errors, configure your Typesense server to allow your domain.
  • Firewall: Ensure port 443 (or your custom port) is open on the Typesense server.

Search Bar Not Appearing?

If the standard search bar isn't replaced, go to SwiftSearch Settings and ensure "Override Default Search" is enabled. Alternatively, place the [swift_search] shortcode manually.

Schema Mismatch / Re-Index Required

If you see a "Critical Schema Mismatch" error, it means your data structure has changed (e.g., you added a new Custom Field or upgraded the plugin).

Solution: Simply click the Index All button in the plugin settings to rebuild the collection.
